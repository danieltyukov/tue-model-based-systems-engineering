11. ElevatorÔÉÅ

In this exercise we will build a plant model for an elevator. To test the plant model, a manual controller with SVG drawing and SVG interface file is given to be downloaded. The elevator operates in a building with three floors. Each floor has buttons to go up or down.

The only communication between the plant (model) and the controller takes place via actuator and sensor signals.

The actuators in the elevator consist of up and down controls of the lift motor, and an actuator for opening and closing the doors. If the motor actuator of the doors is set to off, the doors will open.

There is a sensor on the elevator which indicates when the elevator has reached a floor. Another sensor indicates when the doors of the elevator are fully closed. Furthermore, there are sensors that indicate when a button is pressed at one of the floors, or in the elevator itself. Below is a list of all the actuators and sensors and their functionality.

To help you in getting started, the following five files are given:

Download File

Description

SVG drawing of elevator visualization

elevator.tooldef

Tooldef file

Running the tooldef file will run the elevator visualization with the following behavior:

  • Each of the request buttons and actuators can be pressed.

  • Activation of the buttons or sensors causes them to turn green.

  • The request buttons are automatically deactivated 0.5 seconds after they have been pressed.

  • Pressing the actuators functions as a toggle. After each press, the state of the actuator is reversed.

  • Operation of the elevator and its doors is not yet modelled.

There are two questions to complete for this elevator exercise:

  1. Run the tooldef file and observe that actuators and buttons to change color after having been pressed.

  2. Complete the plant model: model the behaviour of the doors, the vertical movement of the elevator, and the associated sensors. Note that you do not need to provide any additional controller.
