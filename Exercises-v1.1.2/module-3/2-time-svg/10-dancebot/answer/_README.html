Robot dance revolution answer

The SVG interface of the dancing robot can be seen in the dancebot-svg.cif file. The SVG with the corresponding groups can be seen in the dancebot.svg file. The groups arm-left and arm-right consist of all the parts that make up the arm. When the upper body moves the arms have move with it and so does the head, so the goup upper-body consists of the main body part, the head and both the groups for the arms. The same goes for the total body, so that group consists of all previous made groups plus the piston group. To get the body to perform both a translation and a rotation on the body, a nested group is used. In the code, the group body-2 contains nothing but the group body. The dancebot.tooldef file can be used to animate the simulation.

dancebot-svg.cif
svgfile "dancebot.svg";

import "dancebot.cif";

//buttons
svgin id "button-1" event u_dance_1;
svgin id "button-2" event u_dance_2;
svgin id "button-3" event u_dance_3;

svgout id "button-1" attr "fill" value if dancer.dance_1: "green" else "grey" end;
svgout id "button-2" attr "fill" value if dancer.dance_2: "green" else "grey" end;
svgout id "button-3" attr "fill" value if dancer.dance_3: "green" else "grey" end;

//visualisation
svgout id "body-2"     attr "transform" value fmt("rotate(%s, 225, 395)", dancer.a);
svgout id "body"       attr "transform" value fmt("translate(0, %s)", -dancer.l + 100);
svgout id "upper-body" attr "transform" value fmt("rotate(%s, 225, 295)", dancer.b);
svgout id "arm-left"   attr "transform" value fmt("rotate(%s, 180, 200)", dancer.c);
svgout id "arm-right"  attr "transform" value fmt("rotate(%s, 270, 200)", dancer.d);
svgout id "piston"     attr "height"    value dancer.l;
Tooldef script
from "lib:cif" import *;             //Imports the library to run CIF 3 files

cifsim(
   "dancebot-svg.cif",
   "-i svg",                         // Allow user input
   "--frame-rate=20",
);
