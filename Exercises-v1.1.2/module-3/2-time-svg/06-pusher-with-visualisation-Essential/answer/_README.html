Pusher with visualisation answer

Using the uncontrollable event u_pusher_toggle and the automaton a_pusher_button, allows the actuator a_pusher to be pressed, to toggle the value of the actuator. This can be seen in the pusher2.cif file. The corresponding visualisation of the SVG file is declared by the pusher2-svg.cif file. Using the pusher2.tooldef file the pusher can be simulated.

pusher2.cif
// Boundaries between which the pusher can move
const real MAXX = 4;
const real MINX = 0;
const real VPUSH = 10;

cont x der v;
// The actuator on results in a speed when between given boundaries
alg real v =  if       a_pusher and x    <= MAXX:  VPUSH
              elif not a_pusher and MINX <= x   : -VPUSH
              else 0
              end;

// The actuator is activated in specific time intervals
alg bool a_pusher = a_pusher_button.on;

uncontrollable u_pusher_toggle;

automaton a_pusher_button:
    location off:
        initial;
        edge u_pusher_toggle goto on;

    location on:
        edge u_pusher_toggle goto off;
end
pusher2-svg.cif
import  "pusher2.cif";
svgfile "pusher.svg";

const real SVGfactor = 20;

svgin id "a_pusher" event u_pusher_toggle;


svgout id "g_shaft_head" attr "transform" value fmt("translate(%s,0)", x * SVGfactor);

svgout id "a_pusher"     attr "fill"      value if a_pusher: "green" else "lightgrey" end;
pusher2.tooldef
from "lib:cif" import *;             //Imports the library to run CIF 3 files

cifsim(
   "pusher2-svg.cif",
   "-i svg",
   "--frame-rate=20"
);
